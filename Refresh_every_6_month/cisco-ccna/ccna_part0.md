# Вводный блок CCNA: Основы сетевых технологий

## Введение в компьютерные сети

Добро пожаловать в мир сетевых технологий! Прежде чем погружаться в настройку маршрутизаторов и коммутаторов, важно понять фундаментальные концепции, на которых строятся современные сети.

---

## 1. Модель OSI (Open Systems Interconnection)

Модель OSI — это концептуальная модель, которая стандартизирует функции телекоммуникационной или вычислительной системы в семь уровней.

### Уровни модели OSI (снизу вверх):

**7. Прикладной уровень (Application Layer)**
- Взаимодействие с конечным пользователем
- Протоколы: HTTP, HTTPS, FTP, SMTP, DNS, SSH
- Пример: веб-браузер, почтовый клиент

**6. Уровень представления (Presentation Layer)**
- Преобразование, шифрование и сжатие данных
- Форматы: JPEG, GIF, SSL/TLS
- Обеспечивает, чтобы данные были читаемы принимающей системой

**5. Сеансовый уровень (Session Layer)**
- Управление сеансами связи между приложениями
- Установка, поддержание и завершение соединений
- Примеры: NetBIOS, RPC

**4. Транспортный уровень (Transport Layer)**
- Надежная доставка данных между конечными точками
- Протоколы: **TCP** (надежный) и **UDP** (ненадежный, но быстрый)
- Сегментация данных, контроль потока, исправление ошибок
- Использует **порты** для идентификации приложений

**3. Сетевой уровень (Network Layer)**
- Маршрутизация пакетов между сетями
- Логическая адресация: **IP-адреса** (IPv4, IPv6)
- Протоколы: IP, ICMP, OSPF, EIGRP, BGP
- Устройства: маршрутизаторы (routers)

**2. Канальный уровень (Data Link Layer)**
- Передача данных между соседними узлами в одной сети
- Физическая адресация: **MAC-адреса**
- Подуровни: LLC и MAC
- Протоколы: Ethernet, Wi-Fi (802.11), PPP
- Устройства: коммутаторы (switches), мосты

**1. Физический уровень (Physical Layer)**
- Передача битов по физической среде
- Кабели (медные, оптоволоконные), радиоволны
- Электрические сигналы, световые импульсы
- Устройства: концентраторы (hubs), повторители

### Мнемоническое правило
**Снизу вверх:** Please Do Not Throw Sausage Pizza Away  
**Сверху вниз:** All People Seem To Need Data Processing

## 2. Модель TCP/IP

Практическая модель, используемая в интернете. Состоит из 4 уровней:

1. **Прикладной** (Application) — HTTP, FTP, DNS, SSH
2. **Транспортный** (Transport) — TCP, UDP
3. **Интернет** (Internet) — IP, ICMP, ARP
4. **Сетевого доступа** (Network Access) — Ethernet, Wi-Fi

### Сравнение OSI и TCP/IP

```
OSI                    TCP/IP
-------------------    -------------------
Application    ]       
Presentation   ] ----  Application
Session        ]       
Transport      ----    Transport
Network        ----    Internet
Data Link      ]       
Physical       ] ----  Network Access
```

## 3. Инкапсуляция данных

Процесс добавления заголовков на каждом уровне при передаче данных:

**Данные → Сегмент (L4) → Пакет (L3) → Кадр (L2) → Биты (L1)**

- **Данные** (Application): исходное сообщение
- **Сегмент** (Transport): добавляется TCP/UDP заголовок с портами
- **Пакет** (Network): добавляется IP заголовок с адресами источника и назначения
- **Кадр** (Data Link): добавляется Ethernet заголовок с MAC-адресами
- **Биты** (Physical): преобразование в электрические/световые сигналы

При получении происходит **декапсуляция** — обратный процесс.

## 4. IP-адресация

### IPv4
- Формат: 32 бита (4 октета), например: **192.168.1.1**
- Классы адресов: A, B, C, D, E
- Частные диапазоны (RFC 1918):
  - Class A: 10.0.0.0/8
  - Class B: 172.16.0.0/12
  - Class C: 192.168.0.0/16

### Маска подсети
Определяет, какая часть адреса относится к сети, а какая к хосту.
- Пример: 255.255.255.0 или /24
- **Сетевая часть + Хостовая часть = IP-адрес**

### IPv6
- Формат: 128 бит, например: **2001:0db8:85a3::8a2e:0370:7334**
- Решает проблему исчерпания адресов IPv4

---

## 5. MAC-адреса

- Физический адрес сетевой карты
- Формат: 48 бит (6 байт в шестнадцатеричном формате)
- Пример: **00:1A:2B:3C:4D:5E**
- Первые 3 байта — OUI (идентификатор производителя)
- Последние 3 байта — уникальный номер устройства

## 6. Основные сетевые устройства

### Хаб (Hub) — Уровень 1
- Простой повторитель сигнала
- Отправляет данные на все порты (broadcast)
- Создает коллизионный домен
- Устаревшее устройство

### Коммутатор (Switch) — Уровень 2
- Использует таблицу MAC-адресов
- Отправляет данные только на нужный порт
- Разделяет коллизионные домены
- Основное устройство локальных сетей

### Маршрутизатор (Router) — Уровень 3
- Маршрутизация между разными сетями
- Использует таблицу маршрутизации и IP-адреса
- Разделяет broadcast-домены
- Принимает решения на основе сетевых адресов

---

## 7. Протоколы транспортного уровня

### TCP (Transmission Control Protocol)
- **Ориентирован на соединение**
- Гарантирует доставку данных
- Контроль потока и исправление ошибок
- Three-way handshake: SYN → SYN-ACK → ACK
- Использование: HTTP, HTTPS, FTP, SSH, Email

### UDP (User Datagram Protocol)
- **Без установления соединения**
- Не гарантирует доставку
- Быстрее, меньше накладных расходов
- Использование: DNS, DHCP, VoIP, потоковое видео, онлайн-игры

### Порты
- Числа от 0 до 65535
- **Well-known ports** (0-1023): HTTP(80), HTTPS(443), SSH(22), FTP(21), DNS(53)
- **Registered ports** (1024-49151)
- **Dynamic/Private ports** (49152-65535)

---

## 8. Основные протоколы

**ARP (Address Resolution Protocol)**
- Преобразует IP-адрес в MAC-адрес
- Работает на канальном уровне

**ICMP (Internet Control Message Protocol)**
- Диагностические сообщения (ping, traceroute)
- Сообщения об ошибках

**DHCP (Dynamic Host Configuration Protocol)**
- Автоматическое назначение IP-адресов
- Выдает IP, маску, шлюз, DNS

**DNS (Domain Name System)**
- Преобразует доменные имена в IP-адреса
- Иерархическая распределенная база данных

---

## 9. Типы передачи данных

**Unicast** — от одного к одному  
**Broadcast** — от одного ко всем в сети  
**Multicast** — от одного к группе подписчиков

## 10. Процесс коммуникации (пример)

Когда вы открываете веб-сайт:

1. **Уровень 7**: Браузер формирует HTTP-запрос
2. **Уровень 4**: TCP добавляет порт источника и назначения (80/443)
3. **Уровень 3**: IP добавляет адреса источника и назначения
4. **Уровень 2**: Ethernet добавляет MAC-адреса
5. **Уровень 1**: Данные преобразуются в сигналы и передаются по кабелю

На каждом устройстве происходит обработка на соответствующих уровнях:
- **Коммутатор**: проверяет MAC-адреса (L2)
- **Маршрутизатор**: проверяет IP-адреса и маршрутизирует (L3)
- **Сервер**: обрабатывает весь стек до приложения
